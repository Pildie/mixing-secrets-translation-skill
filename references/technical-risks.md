# Technical Risks Analysis

| Risk | Severity | Description | Mitigation |
| :--- | :--- | :--- | :--- |
| **Phase vs Polarity Confusion** | CRITICAL | Translating both as "inversão de fase". | Enforce: polarity inversion = inversão de polaridade. |
| **Literal Timbre Translation** | HIGH | Translating "Mud" as "Lama" or "Warmth" as "Mornura". | Use market terms: "empastamento" and "calor". |
| **Academic Formalization** | MEDIUM | Turning colloquial phrases into thesis-style writing. | Read output aloud; ensure it sounds like a studio conversation. |
| **UI Localization** | HIGH | Translating buttons like "Threshold" to "Limiar". | Keep UI labels in English as they appear in software. |
| **Buss vs Bus** | LOW | Losing the author's preferred "buss" spelling. | Maintain "buss" for technical terms/labels. |
| **Phonetic Transliteration** | MEDIUM | Writing "kei-fator" for "Q factor". | Use "fator Q". |
